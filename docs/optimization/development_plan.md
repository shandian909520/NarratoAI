# NarratoAI 业务优化开发方案

## 1. 概述

### 1.1 项目背景

NarratoAI 是一款 AI 影视解说与自动化剪辑工具，采用 Python 3.12+、Streamlit、LiteLLM、MoviePy、FFmpeg 等技术栈构建。本文档针对产品功能优化与业务拓展制定详细开发方案。

### 1.2 优化目标

- **用户体验优化**：提升批量处理能力、可视化程度与交互体验
- **内容质量优化**：增强 AI 生成内容的质量与多样性
- **变现功能拓展**：构建可持续的商业化路径
- **运营效率提升**：降低用户操作成本，提高复用率

### 1.3 模块总览

| 模块 | 包含功能 | 优先级 |
|------|----------|--------|
| 用户体验优化 | 批量处理、进度可视化、预览功能、模板系统 | P0 |
| 内容质量优化 | 智能封面生成、热门BGM推荐、多音轨解说、情绪配乐 | P1 |
| 变现功能拓展 | 会员体系、API开放、模板交易市集 | P1 |
| 运营效率提升 | 草稿保存、历史记录、项目管理、数据统计 | P0 |

## 2. 功能详细规划

### 2.1 用户体验优化

#### 2.1.1 批量处理

**功能描述**

支持用户同时管理多个视频处理任务，实现队列化处理，提高大规模内容生产效率。

**技术方案**

- **视频队列管理**：基于 SQLite 构建任务队列数据库，支持任务的创建、暂停、恢复、取消
- **批量任务调度**：使用 APScheduler 实现定时调度，结合 ThreadPoolExecutor 实现并发控制
- **队列状态持久化**：任务状态、进度、中间结果存储至数据库，支持服务重启后恢复

**涉及文件**

| 文件路径 | 说明 |
|----------|------|
| `app/services/queue_manager.py` | 队列管理核心服务 |
| `app/services/task_scheduler.py` | 任务调度服务 |
| `app/database/queue_db.py` | 队列相关数据库模型 |
| `app/ui/batch_processing.py` | 批量处理 UI 组件 |

**优先级**：P0

**预期工时**：5-6 人天

---

#### 2.1.2 进度可视化

**功能描述**

提供多步骤进度条、实时状态更新、预估完成时间等增强体验。

**技术方案**

- **多步骤进度条**：基于 Streamlit 的自定义组件，分阶段展示（解析→文案生成→配音→剪辑→导出）
- **实时状态更新**：使用 WebSocket 或 SSE (Server-Sent Events) 推送实时进度
- **预估时间计算**：基于历史任务耗时统计，动态计算剩余时间

**涉及文件**

| 文件路径 | 说明 |
|----------|------|
| `app/ui/progress_bar.py` | 进度条组件 |
| `app/services/progress_tracker.py` | 进度追踪服务 |
| `app/services/realtime_updates.py` | 实时更新服务 |

**优先级**：P0

**预期工时**：3-4 人天

**功能描述**

**技术方案**

## 4. 技术注意事项

### 4.1 数据库设计

- 使用 SQLite 作为本地存储，配合 SQLAlchemy ORM
- 关键表需添加索引（任务表、用户表、模板表）
- 定期归档历史数据，避免数据库膨胀

### 4.2 并发处理

- 任务调度使用线程池控制并发数（建议 2-4 个）
- UI 响应与后台任务分离，避免阻塞
- 使用 asyncio 优化 IO 密集型操作

### 4.3 缓存策略

- TTS 生成结果缓存至本地，避免重复调用
- 音乐库索引缓存至内存，加快检索
- 模板配置缓存，减少文件 IO

### 4.4 错误处理

- 任务失败自动重试（最多 3 次，指数退避）
- 关键步骤设置检查点，失败后可从检查点恢复
- 详细错误日志记录，便于问题排查

### 4.5 资源管理

- 视频素材处理后及时清理临时文件
- 定期检查磁盘空间，避免存储溢出
- 大文件处理使用流式IO，避免内存占用过高

### 4.6 API 安全

- 所有外部 API 调用添加超时限制（建议 30s）
- 敏感配置使用环境变量，避免硬编码
- 定期轮换 API Key

### 4.7 Streamlit 特定

- 使用 `st.session_state` 管理状态
- 长时间运行操作使用 `st.progress` + `st.empty`
- 避免在循环中频繁调用 Streamlit 组件
